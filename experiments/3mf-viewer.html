How this works

About

Parses a .3mf file entirely in the browser — nothing is uploaded. Renders the mesh with WebGL and surfaces textual metadata embedded in the file such as title, designer, application, dimensions, and triangle count.

How to use

Drop a .3mf file onto the drop zone or click to browse. Drag to orbit the model; scroll or pinch to zoom. Hit Reset view to return to the default angle.

Technical details

A .3mf file is a ZIP archive (with ZIP64 support for large files). The Central Directory is located via the PK\x05\x06 end record (falling back to the ZIP64 PK\x06\x06 record), and entries are decompressed with the browser's DecompressionStream API (deflate-raw mode).

Many slicers (Bambu Studio, MakerWorld) use the 3MF Production Extension: the top-level 3dmodel.model contains only component references while the actual geometry lives in 3D/Objects/*.model. To handle this transparently, every .model file in the archive is parsed and their geometry combined. DOMParser + getElementsByTagNameNS('*', ...) handles namespace prefixes safely.

Vertex normals are computed by accumulating cross-product face normals per vertex and normalizing. The WebGL 1 renderer uses ambient + diffuse lighting with Euler-angle orbit controls. Large meshes (over 65 536 vertices) use the OES_element_index_uint extension for 32-bit indices.
